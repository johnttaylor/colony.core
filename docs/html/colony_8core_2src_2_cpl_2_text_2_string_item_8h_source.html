24 /** This paritally concrete provides a wrapper class that allows a String
25  object to be stored directly in one the CPL Containers. A StringItem
26  can be stored in a SList, DList, Dictionary, and Map.
27 
28  Notes:
29 
30  o The allocation of the actual String is deferred to child classes.
31  o When storing a StringItem in a 'keyed' container (i.e. Dictionary
32  and Map) the application MUST NOT change the value of the internal
33  string since it is the 'key' value that is used to determine its
34  placement in the Dictionary/Map. Modifying the internal String while
35  the StringItem is in a Diciontary/Map will CORRUPT the Container.
36  o StringItem instances CANNOT be copied. This is to avoid potential
37  Container corruption since the individual Items contain the memory
38  for the Containers' linkage, i.e. a copied container Item is
39  analogous to storing a single Item in more than one Container at the
40  same time.
41  */
