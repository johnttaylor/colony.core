52  /** Constructor. The amount of storage initially allocated for the string is
53  the maximum of the size of the source string AND the value of initialSize.
54  Whatever value is choosen, it is then rounded up to the nearest blocksize
55  multiple. There are two main reasons to specify an intialSize. 1) To
56  reduce the number of malloc/free that occur over the life of the String.
57  If you supply a large enough initialSize - no additional mallocs will be
58  needed. 2) If you plan to use the format(..) methods. The format()
59  methods will NOT allocate additional storage. To prevent the format()
60  methods from truncating, you must start with a large enough 'buffer'.
61  The blocksize parameter can be used to control the size of the 'chunks'
62  memory is allocated in. This 'blocking' paradigm helps to reduce
63  fragmentation and number of internal malloc/free operations.
64 
65  NOTE: The space reserved for the trailing null terminator is part of the
66  block calculation, i.e. if your initial size is 16, and your
67  blocksize is 16, then number of bytes allocated is 32 to allow
68  space for a 16 byte string and a one byte terminator and then
69  rounded up to the next block size.
70  */
