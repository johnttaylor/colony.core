25 /** This concrete class implements a blocking Transmit Stream UART driver. What
26  does 'blocking Transmit Stream' mean? 'Blocking' means that the calling
27  thread will block when the transmit buffer is full (and will be unblocked
28  once there is free space in the transmit buffer). 'Stream' means that data
29  is transmitted as a continual stream of bytes, i.e. the data is NOT broken
30  down into frames, packets, etc. (think C stdout stream).
31 
32  This driver ASSUMES that at most there is only ONE client attempt to use the
33  driver at any given time. It is okay to call the start(), stop(), write()
34  from different threads - but the calls CANNOT be concurrent. It is the
35  application's responsibility to provide additional
36  thread-safety/concurrence protection.
37 
38  NOTE: Not all UART behave the same with respect to transmitting the 'first'
39  byte (in a stream/buffer of bytes). One method (which is the default
40  for Cpl) is that a 'TX-done' interrupt is generated any time the
41  TX data register is empty and the TX-done interrupt is enabled, i.e.
42  the first byte in the stream/buffer is handle the same as all of the
43  other bytes. The second method is that the application/driver is
44  required to expclity write the first byte to the TX Data register,
45  i.e. no interrupt for the intitial byte.
46 
47  */
