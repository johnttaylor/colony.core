Cpl::Timer::Local< CONTEXT > Class Template Reference

Detailed Description

template<class CONTEXT>
class Cpl::Timer::Local< CONTEXT >

This template class implements a Local Timer that is context independent and allows for a single context to contain many Timers.

A Local timer is defined as timer that has its tick source in the SAME thread. Because the timer methods and callback all occur in the the same thread, the CONTEXT will never receive a timer expired callback AFTER the timer's stop() method has been called.

NOTES: 

o Because the timing source of an individual thread may NOT be a 
  clean divider of the timer duration, the timer duration is taken
  as the minimum.  For example: if the timing source has a resolution
  of 20msec per count, and the timer duration on the start() timer
  call is 5 msec, then the timer will expire after the next full count,
  i.e. after 20msec, not 5msec.  IT IS THE APPLICATION'S RESPONSIBILITY
  TO MANAGE THE RESOLUTION OF THE TIMING SOURCES.

Template args: CONTEXT Type of the Class that implements the context for the timer
