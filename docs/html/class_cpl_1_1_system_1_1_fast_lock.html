Cpl::System::FastLock Class Reference

Detailed Description

This class defines an interface for a "Fast Lock". A Fast Lock provides mutual exclusion and/or a critical section protection similiar to a Mutex except that is it intended to be lighter weight and/or have faster performance than a traditional Mutex. However, the performance gain comes with the following constraints: 

o Non recursive semantics.  The calling thread CANNOT attemp to acquire
  the lock a second time once it has already acquire the lock. 

o The code that is protected by this lock MUST BE VERY SHORT time 
  wise and NOT call an operating system methods.

Why the above the constraints? The implementation of a FastLock is target platform specific and the implemenation can vary from disable/enable interrupts on a RTOS to spin-locks in user space. So, use FastLocks with care and always honor the above constraints.

void Cpl::System::FastLock::lock ( void  )

This method is invoked prior to entering a critical section. If another thread currently "owns" the mutex, the current thread will "wait" until it can obtain ownership before proceeding.
