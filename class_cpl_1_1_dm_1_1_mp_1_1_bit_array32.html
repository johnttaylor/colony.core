Cpl::Dm::Mp::BitArray32 Class Reference

This template class provides a concrete implementation for a Point who's data is a a bit array of N bits. More...

Detailed Description

This template class provides a concrete implementation for a Point who's data is a a bit array of N bits.

The underlying storage of the bit array is 32 bit unsigned integers. A side effect of this storage mechanism the bit ordering in the JSON 'val' string is dependent on the target platform's Endian architecture.

The toJSON()/fromJSON format is:

{ name:"<mpname>", type:"<mptypestring>", valid:true|false, seqnum:nnnn, locked:true|false, val:"<bits>" }
where <bits> is a string of N digits ('1' or '0') where the left most digit is
is the MSb of byte[0] and the right most digit is the LSb of byte[N].
Whether byte[0] is the MSB or LSB is dependent on the big/little Endian
architecture of the target platform.
For example a 16bit Array (as binary hex: dataword[0]=0x30, dataword[1]=0x09, dataword[2]=0x30, dataword[3]=0x09)
val:"0011000000001001"

NOTE: All methods in this class ARE thread Safe unless explicitly documented otherwise.
